1. Executive Summary
This document provides a complete developer-facing specification of the Al-Nassim Master Catalog
workbook — the central product information management (PIM) artifact for Al-Nassim, a Kuwait-based
household goods distributor serving GCC retail channels including Amazon, Noon, Carrefour, Lulu, IKEA,
Ace Hardware, and Home Centre.
The workbook contains 2,546 products across 63 sheets, with the Master Catalog sheet as the primary
data table. This documentation enables another developer to fully understand the workbook structure,
recreate its behavior, and integrate with it without needing to inspect the original file.

2. Workbook Architecture Overview
The Al-Nassim Master Catalog workbook is a multi-sheet Excel file containing 63 sheets organized into
three functional layers:
Layer 1: Core Data (1 sheet)
 Master Catalog — the primary product table with 2,546 rows × 52 columns. This is the only sheet
that should be directly edited by catalog managers.
Layer 2: Reference & Taxonomy (6 sheets)
 Taxonomy Reference — defines the valid Department → Category → Subcategory hierarchy (21
entries).
 Inferred Classifications — 208 products whose classification was inferred (not direct-matched) during
the V2 taxonomy project, with reviewer actions.
 Classification Summary — roll-up of the V2 classification coverage (100% achieved).
 Validation Report — taxonomy validation results.
 Translation Report — translation pipeline summary.
 Name Rewrite Report / Report V2 — summaries of the name-rewrite passes.
Layer 3: Audit & Correction Logs (56 sheets)
These sheets document the 8 rounds of translation quality assurance (V3 through V8, plus override
batches). Each audit round has a paired Audit sheet (findings) and Corrections sheet (applied fixes). Key
audit sheets include:
 Translation Audit V3 — the master audit log with 2,546 rows, one per product, showing action
(UNCHANGED / CORRECTED / FLAG_REVIEW / APPLIED) and reason.
 Translation Audit Summary V3 — roll-up stats and top recurring issues.
 Name Rewrite Log V2 — 935 rows documenting phrase-first translation rewrites.
 Audit Change Log — 768 individual changes from the V2 audit.
 Translation Quality Corrections — 472 corrections from the quality audit.
 Cross-Validation Discrepancies V3 — 517 cross-validation findings.
 Semantic Translation Corrections — 402 semantic fixes.
 Translation QA Corrections — 1,413 corrections from the final QA pass.
 Complete Re-Audit Corr. — 474 corrections from the complete re-audit.
 Completeness Corrections — 373 corrections filling missing attributes.
 V5 Application Summary — 226 FLAG_REVIEW rows resolved.
 V6 Refinement Log — 686 rows where Product Type was upgraded from 'Other' to specific types.
 V7 Override Log — 115 rows where forbidden phrases were replaced per the Override Lexicon.

Design Rationale
 Ordering: Identity first, then classification, then descriptions, then specs, then commercial, then SEO,
then internal QA. This mirrors the order in which a catalog manager reads a product record.
 Bilingual pairing: Bilingual pairs (Name AR/EN, Color AR/EN, Material AR/EN, SEO Title AR/EN, etc.)
are always adjacent to facilitate side-by-side review.
 Spec grouping: All dimension columns (Length, Width, Height, Diameter) are grouped together,
followed by their shared Dimension Unit.
 QA isolation: Internal/QA columns (Validation Status, Confidence Score) are at the far right so they
don't interfere with customer-facing data views.
