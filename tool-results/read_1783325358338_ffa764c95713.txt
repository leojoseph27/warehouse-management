    74→2. Workbook Architecture Overview
    75→The Al-Nassim Master Catalog workbook is a multi-sheet Excel file containing 63 sheets organized into
    76→three functional layers:
    77→Layer 1: Core Data (1 sheet)
    78→ Master Catalog — the primary product table with 2,546 rows × 52 columns. This is the only sheet
    79→that should be directly edited by catalog managers.
    80→Layer 2: Reference & Taxonomy (6 sheets)
    81→ Taxonomy Reference — defines the valid Department → Category → Subcategory hierarchy (21
    82→entries).
    83→ Inferred Classifications — 208 products whose classification was inferred (not direct-matched) during
    84→the V2 taxonomy project, with reviewer actions.
    85→ Classification Summary — roll-up of the V2 classification coverage (100% achieved).
    86→ Validation Report — taxonomy validation results.
    87→ Translation Report — translation pipeline summary.
    88→ Name Rewrite Report / Report V2 — summaries of the name-rewrite passes.
    89→Layer 3: Audit & Correction Logs (56 sheets)
    90→These sheets document the 8 rounds of translation quality assurance (V3 through V8, plus override
    91→batches). Each audit round has a paired Audit sheet (findings) and Corrections sheet (applied fixes). Key
    92→audit sheets include:
    93→ Translation Audit V3 — the master audit log with 2,546 rows, one per product, showing action
    94→(UNCHANGED / CORRECTED / FLAG_REVIEW / APPLIED) and reason.
    95→ Translation Audit Summary V3 — roll-up stats and top recurring issues.
    96→ Name Rewrite Log V2 — 935 rows documenting phrase-first translation rewrites.
    97→ Audit Change Log — 768 individual changes from the V2 audit.
    98→ Translation Quality Corrections — 472 corrections from the quality audit.
    99→ Cross-Validation Discrepancies V3 — 517 cross-validation findings.
   100→ Semantic Translation Corrections — 402 semantic fixes.
   101→ Translation QA Corrections — 1,413 corrections from the final QA pass.
   102→ Complete Re-Audit Corr. — 474 corrections from the complete re-audit.
   103→ Completeness Corrections — 373 corrections filling missing attributes.

   105→ V6 Refinement Log — 686 rows where Product Type was upgraded from 'Other' to specific types.
   106→ V7 Override Log — 115 rows where forbidden phrases were replaced per the Override Lexicon.

   167→Design Rationale
   168→ Ordering: Identity first, then classification, then descriptions, then specs, then commercial, then SEO,
   169→then internal QA. This mirrors the order in which a catalog manager reads a product record.
   170→ Bilingual pairing: Bilingual pairs (Name AR/EN, Color AR/EN, Material AR/EN, SEO Title AR/EN, etc.)
   171→are always adjacent to facilitate side-by-side review.
   172→ Spec grouping: All dimension columns (Length, Width, Height, Diameter) are grouped together,
   173→followed by their shared Dimension Unit.
   174→ QA isolation: Internal/QA columns (Validation Status, Confidence Score) are at the far right so they
   175→don't interfere with customer-facing data views.
