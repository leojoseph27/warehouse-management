    71→2. Workbook Architecture Overview
    72→The Al-Nassim Master Catalog workbook is a multi-sheet Excel file containing 63 sheets organized into
    73→three functional layers:
    74→Layer 1: Core Data (1 sheet)
    75→ Master Catalog — the primary product table with 2,546 rows × 52 columns. This is the only sheet
    76→that should be directly edited by catalog managers.
    77→Layer 2: Reference & Taxonomy (6 sheets)
    78→ Taxonomy Reference — defines the valid Department → Category → Subcategory hierarchy (21
    79→entries).
    80→ Inferred Classifications — 208 products whose classification was inferred (not direct-matched) during
    81→the V2 taxonomy project, with reviewer actions.
    82→ Classification Summary — roll-up of the V2 classification coverage (100% achieved).
    83→ Validation Report — taxonomy validation results.
    84→ Translation Report — translation pipeline summary.
    85→ Name Rewrite Report / Report V2 — summaries of the name-rewrite passes.
    86→Layer 3: Audit & Correction Logs (56 sheets)
    87→These sheets document the 8 rounds of translation quality assurance (V3 through V8, plus override
    88→batches). Each audit round has a paired Audit sheet (findings) and Corrections sheet (applied fixes). Key
    89→audit sheets include:
    90→ Translation Audit V3 — the master audit log with 2,546 rows, one per product, showing action
    91→(UNCHANGED / CORRECTED / FLAG_REVIEW / APPLIED) and reason.
    92→ Translation Audit Summary V3 — roll-up stats and top recurring issues.
    93→ Name Rewrite Log V2 — 935 rows documenting phrase-first translation rewrites.
    94→ Audit Change Log — 768 individual changes from the V2 audit.
    95→ Translation Quality Corrections — 472 corrections from the quality audit.
    96→ Cross-Validation Discrepancies V3 — 517 cross-validation findings.
    97→ Semantic Translation Corrections — 402 semantic fixes.
    98→ Translation QA Corrections — 1,413 corrections from the final QA pass.
    99→ Complete Re-Audit Corr. — 474 corrections from the complete re-audit.
   100→ Completeness Corrections — 373 corrections filling missing attributes.
   101→ V5 Application Summary — 226 FLAG_REVIEW rows resolved.
   102→ V6 Refinement Log — 686 rows where Product Type was upgraded from 'Other' to specific types.
   103→ V7 Override Log — 115 rows where forbidden phrases were replaced per the Override Lexicon.

   161→Design Rationale
   162→ Ordering: Identity first, then classification, then descriptions, then specs, then commercial, then SEO,
   163→then internal QA. This mirrors the order in which a catalog manager reads a product record.
   164→ Bilingual pairing: Bilingual pairs (Name AR/EN, Color AR/EN, Material AR/EN, SEO Title AR/EN, etc.)
   165→are always adjacent to facilitate side-by-side review.
   166→ Spec grouping: All dimension columns (Length, Width, Height, Diameter) are grouped together,
   167→followed by their shared Dimension Unit.
   168→ QA isolation: Internal/QA columns (Validation Status, Confidence Score) are at the far right so they
   169→don't interfere with customer-facing data views.
